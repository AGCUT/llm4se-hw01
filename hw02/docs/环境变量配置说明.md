# 环境变量配置说明

## 前端环境变量

在 `frontend` 目录下创建 `.env.local` 文件，内容如下：

```bash
# API配置
VITE_API_URL=http://localhost:3000
VITE_WS_URL=ws://localhost:3000

# Supabase配置（必须配置！）
# 从 Supabase Dashboard → Settings → API 获取
VITE_SUPABASE_URL=https://xxxxx.supabase.co
VITE_SUPABASE_ANON_KEY=eyJhbGc...你的anon_key

# 高德地图（可选）
VITE_AMAP_KEY=your_amap_key_here

# 科大讯飞语音（可选）
VITE_XUNFEI_APP_ID=your_xunfei_app_id
VITE_XUNFEI_API_KEY=your_xunfei_api_key
```

## 获取 Supabase 配置

1. 登录 [Supabase](https://supabase.com)
2. 进入您的项目
3. 点击 **Settings** → **API**
4. 复制以下信息：
   - **Project URL** → 填入 `VITE_SUPABASE_URL`
   - **anon public key** → 填入 `VITE_SUPABASE_ANON_KEY`

## 重要提示

⚠️ **安全警告：**
- ✅ 只使用 `anon public` key，不要使用 `service_role` key
- ✅ `.env.local` 文件不会被提交到 Git（已在 .gitignore 中）
- ✅ 确保 Supabase RLS（行级安全）已启用

## 配置完成后

重启开发服务器：

```bash
# 停止当前服务器（Ctrl+C）
# 然后重新启动
npm run dev
```
