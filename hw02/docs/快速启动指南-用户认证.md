# 🚀 快速启动指南 - 用户认证功能

> 5分钟快速启动并测试用户认证功能

## 📋 前提条件

✅ 已创建 Supabase 项目  
✅ 已执行数据库 SQL（`docs/supabase数据库设计.sql`）  
✅ 已获取 Supabase URL 和 ANON_KEY

---

## 🎯 启动步骤

### 步骤 1: 配置环境变量（2分钟）

在 `frontend` 目录创建 `.env.local` 文件：

```bash
cd frontend
```

创建 `.env.local` 文件，内容如下：

```env
VITE_SUPABASE_URL=https://xxxxx.supabase.co
VITE_SUPABASE_ANON_KEY=eyJhbGc...你的完整key
```

**获取配置的位置：**
1. 登录 Supabase Dashboard
2. 选择你的项目
3. 点击 **Settings** → **API**
4. 复制 **Project URL** 和 **anon public** key

### 步骤 2: 安装依赖（1分钟）

```bash
# 在 frontend 目录下
npm install
```

### 步骤 3: 启动开发服务器（1分钟）

```bash
npm run dev
```

等待启动完成，会显示：

```
VITE v5.0.0  ready in 1234 ms

➜  Local:   http://localhost:5173/
```

### 步骤 4: 测试功能（1分钟）

打开浏览器访问：

1. **首页**: http://localhost:5173
2. **注册页面**: http://localhost:5173/register
3. **登录页面**: http://localhost:5173/login

## 🐛 遇到问题？

### 问题 1: 启动报错 "Missing Supabase environment variables"

**解决:**
- 检查 `.env.local` 文件是否存在
- 检查变量名是否正确（必须以 `VITE_` 开头）
- **重启开发服务器**（环境变量修改后必须重启）

### 问题 2: 注册成功但无法登录

**原因:** Supabase 默认需要邮箱验证

**解决方案 A（临时开发）:**
1. Supabase Dashboard → Authentication → Policies
2. 取消勾选 "Enable email confirmations"

**解决方案 B（生产环境）:**
1. 检查邮箱（包括垃圾箱）
2. 点击验证链接
3. 返回登录

### 问题 3: 表单提交没反应

**检查:**
1. 打开浏览器控制台（F12）
2. 查看是否有错误信息
3. 检查网络请求是否发送
4. 检查 Supabase URL 和 KEY 是否正确

### 问题 4: 路由守卫不工作

**解决:**
1. 清除浏览器 LocalStorage
2. 刷新页面
3. 重新登录

## 💡 开发提示

### 在代码中使用认证

#### 获取当前用户
```tsx
import { useAuth } from '@/hooks/useAuth'

function MyComponent() {
  const { user, profile, isAuthenticated } = useAuth()
  
  return (
    <div>
      {isAuthenticated ? (
        <p>欢迎，{profile?.username}</p>
      ) : (
        <p>请登录</p>
      )}
    </div>
  )
}
```

#### 登出
```tsx
import { useAuthStore } from '@/store/authStore'

function LogoutButton() {
  const { signOut } = useAuthStore()
  
  const handleLogout = async () => {
    await signOut()
    // 自动重定向到登录页
  }
  
  return <button onClick={handleLogout}>退出</button>
}
```
