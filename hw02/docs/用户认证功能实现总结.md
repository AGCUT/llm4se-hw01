## 💻 代码示例

### 在组件中使用认证

#### 方式 1: 使用 Hook

```tsx
import { useAuth } from '@/hooks/useAuth'

function MyComponent() {
  const { user, profile, loading, isAuthenticated } = useAuth()

  if (loading) return <div>加载中...</div>
  if (!isAuthenticated) return <div>请先登录</div>

  return (
    <div>
      <h1>欢迎，{profile?.username || user?.email}</h1>
    </div>
  )
}
```

#### 方式 2: 使用 Store

```tsx
import { useAuthStore } from '@/store/authStore'

function MyComponent() {
  const { user, signOut } = useAuthStore()

  const handleLogout = async () => {
    await signOut()
  }

  return (
    <div>
      <p>用户: {user?.email}</p>
      <button onClick={handleLogout}>退出</button>
    </div>
  )
}
```

#### 方式 3: 直接调用 API

```tsx
import { signInWithEmail } from '@/api/auth.api'

async function login(email: string, password: string) {
  try {
    const { user, session } = await signInWithEmail(email, password)
    console.log('登录成功:', user)
  } catch (error) {
    console.error('登录失败:', error)
  }
}
```

## 🐛 常见问题及解决方案

### Q1: 注册后无法登录

**问题:** Supabase 默认需要邮箱验证

**解决方案:**
1. 检查邮箱（包括垃圾箱）
2. 点击验证链接
3. 或在 Supabase Dashboard 禁用邮箱验证：
   - Authentication → Policies
   - 取消勾选 "Enable email confirmations"

### Q2: 环境变量不生效

**问题:** `.env.local` 文件配置后仍然报错

**解决方案:**
1. 确认文件名为 `.env.local`（不是 `.env`）
2. 确认变量名以 `VITE_` 开头
3. **重启开发服务器**（Ctrl+C 然后 `npm run dev`）

### Q3: 路由守卫不工作

**问题:** 未登录也能访问受保护的页面

**解决方案:**
1. 检查 `App.tsx` 中是否调用了 `refreshAuth()`
2. 检查 `router.tsx` 中路由配置是否正确
3. 清除浏览器 LocalStorage 重试

### Q4: 用户资料为空

**问题:** 登录成功但 `profile` 为 null

**原因:** 数据库触发器创建 profile 有延迟

**解决方案:**
```typescript
// 注册后等待一下
await signUp(email, password, username)
await new Promise(resolve => setTimeout(resolve, 1000))
await refreshAuth()
```

### Q5: CORS 错误

**问题:** 请求被 CORS 策略阻止

**解决方案:**
1. Supabase Dashboard → Settings → API
2. 在 "Site URL" 中添加 `http://localhost:5173`
3. 在 "Additional Redirect URLs" 中添加回调 URL
