# 修复登录页面一直加载问?

## 问题描述

1. 首页 `http://127.0.0.1:5173/` 一直显示加载中，无法跳转到登录页面
2. React Router Future Flag Warning 警告

## 问题原因

1. **`refreshAuth` 函数阻塞**：`getCurrentUser()` ?`getCurrentSession()` 可能会超时，导致 `loading` 状态一直为 `true`
2. **`Home.tsx` 依赖 loading 状?*：如?`loading` 一直为 `true`，页面会一直显示加载中
3. **`useAuth` hook 也可能阻?*：使用了 `supabase.auth.getSession()`，可能会超时

## 修复方案

### 1. 优化 `authStore.ts` 中的 `refreshAuth` 函数

- 添加 2 秒超时保?
- 使用 `Promise.race` 避免长时间阻?
- 获取用户资料改为异步执行，不阻塞主流?

### 2. 简?`App.tsx` 中的认证初始?

- 移除复杂的超时处理逻辑
- 直接调用 `refreshAuth`，它内部已经有超时处?

### 3. 优化 `useAuth.ts` hook

- 添加 2 秒超时保?
- 使用 `Promise.race` 避免 `getSession()` 阻塞
- 添加取消标志，避免内存泄?

### 4. 修改 `Home.tsx` 页面

- 添加 3 秒超时机?
- 即使 `loading` ?`true`? 秒后也显示内?
- 提供更好的用户体?

### 5. React Router 警告

- 已在 `router.tsx` 中添?`v7_startTransition: true`
- 可能需要重启开发服务器才能生效

## 修复后的行为

1. **认证初始?*：最多等?2 秒，超时后继续执?
2. **首页加载**：最多显?3 秒加载动画，然后显示内容
3. **用户体验**：不会一直显示加载中，用户可以正常使用页?
